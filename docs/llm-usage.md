# 大模型调用说明

## 是否使用了大模型

有使用。

## 使用位置

本项目中，大模型主要用于以下场景：

1. 协助梳理需求、拆解系统边界和接口契约。
2. 生成和修订部分代码实现，包括接口、页面、测试与文档。
3. 辅助验证实现是否覆盖题目要求，并整理提交材料。

## 使用原则

虽然使用了大模型，但我没有把模型输出直接当作最终结果，而是按下面方式处理：

1. 对接口契约、部署结果、测试结果做本地或实际命令验证。
2. 对关键业务逻辑继续以代码和测试结果为准，而不是只看自然语言解释。
3. 对系统 A / 系统 B 的角色划分，优先依据真实仓库结构、接口和部署地址来判断。

## 风险与控制

大模型带来的主要风险包括：

1. 可能把旧项目错误解释为当前题目的系统 A。
2. 可能给出“看起来合理但实际上未联调”的方案。
3. 可能遗漏编码、环境变量、部署配置等工程细节。

对应控制方式是：

1. 用真实接口调用验证联调链路。
2. 用 `build`、测试、健康检查和部署命令做证据闭环。
3. 对提交材料中的关键结论，只写已经验证过的内容。

## 最终说明

本次提交中，大模型是“辅助开发与验证工具”，不是“替代验证的结论来源”。最终交付内容仍以本地代码、测试结果、接口联调结果和 Vercel 部署结果为准。
